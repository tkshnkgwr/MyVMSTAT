# システム詳細仕様書: MyVMSTAT

本ドキュメントは、カラー表示可能なシステムリソース監視ツールである **MyVMSTAT** の内部設計、動作アルゴリズム、データパースロジック、およびマルチプラットフォーム設計について定義します。

---

## 1. アーキテクチャ概要

`MyVMSTAT` は、仮想メモリ、プロセススケジューリング、CPU実行割合、および絶対時刻タイムスタンプの情報をターミナル上にカラーで一覧表示するCLIユーティリティです。

```text
       +---------------------------------------------------------+
       |               MyVMSTAT CLI アプリケーション             |
       +----------------------------+----------------------------+
                                    |
            +-----------------------+-----------------------+
            | (Linux プラットフォーム)                      | (クロスプラットフォーム)
            v                                               v
   +-------------------+                           +-------------------+
   | /proc ファイルシステム|                           | sysinfo API レイヤー |
   |   - /proc/stat    |                           |   - CPU 全体使用率 |
   |   - /proc/meminfo |                           |   - スワップ使用量 |
   |                   |                           |   - プロセス一覧   |
   +-------------------+                           +-------------------+
```

---

## 2. 監視メトリクスカラム仕様

表示される情報は、Linux標準の `vmstat` のレイアウトを基準とし、`dstat` 風のカラー強調とタイムスタンプ表示を統合した形式をとります。

### カラムの定義とデータ型

| カテゴリ | カラム名 | データ型 | 単位 | 説明 |
| :--- | :--- | :--- | :--- | :--- |
| **procs** | `r` | 整数 | 個 | 実行中で待機状態にあるプロセス数（Run queue） |
| | `b` | 整数 | 個 | 割り込み不可能なスリープ状態にあるプロセス数（Blocked） |
| **memory**| `swpd` | 整数 | MB | 使用中のスワップメモリ容量 |
| | `free` | 整数 | MB | 空き物理メモリ容量 |
| | `buff` | 整数 | MB | バッファ用メモリ容量 |
| | `cache`| 整数 | MB | キャッシュ用メモリ容量 |
| **system**| `in` | 整数 | 回/秒 | 1秒あたりの割り込み回数 (interrupts) |
| | `cs` | 整数 | 回/秒 | 1秒あたりのコンテキストスイッチ回数 (context switches) |
| **cpu** | `us` | 整数 | % | ユーザープロセスの実行割合（user + nice） |
| | `sy` | 整数 | % | システム（カーネル）プロセスの実行割合（system + irq + softirq） |
| | `id` | 整数 | % | CPUアイドル時間の割合 (idle) |
| | `wa` | 整数 | % | I/Oウェイト時間の割合 (iowait) |
| **time** | `timestamp`| 文字列 | - | ローカルの絶対時刻 (`YYYY-MM-DD HH:MM:SS`) |

## 3. Linux ネイティブデータ取得仕様

Linux環境下では、システムオーバーヘッドを最小限に抑えるため、C関数やライブラリを介さず `/proc` 仮想ファイルシステムから直接データをパースします。

### 3.1 CPU 使用率の算出 (`/proc/stat`)
`/proc/stat` の1行目の `cpu` 累積値行を読み込みます。
```text
cpu  22538 310 11925 829532 2110 0 203 0 0 0
```
- **取得するフィールド**: `user(1)`, `nice(2)`, `system(3)`, `idle(4)`, `iowait(5)`, `irq(6)`, `softirq(7)`, `steal(8)`。
- **計算式**:
  $$\text{TotalTicks} = \text{user} + \text{nice} + \text{system} + \text{idle} + \text{iowait} + \text{irq} + \text{softirq} + \text{steal}$$
  $$\Delta\text{Total} = \text{TotalTicks}_{\text{new}} - \text{TotalTicks}_{\text{old}}$$
  $$\%us = \frac{\Delta(user + nice)}{\Delta\text{Total}} \times 100$$
  $$\%sy = \frac{\Delta(system + irq + softirq)}{\Delta\text{Total}} \times 100$$
  $$\%wa = \frac{\Delta(iowait)}{\Delta\text{Total}} \times 100$$
  $$\%id = \frac{\Delta(idle)}{\Delta\text{Total}} \times 100$$

### 3.2 メモリ使用量のパース (`/proc/meminfo`)
毎サンプリング時に以下の項目をパースし、KBからMBへ変換します（`値 / 1024`）：
- `MemFree` $\rightarrow$ `free` カラム
- `Buffers` $\rightarrow$ `buff` カラム
- `Cached` $\rightarrow$ `cache` カラム
- `SwapTotal` - `SwapFree` $\rightarrow$ `swpd` カラム

### 3.3 割り込みとコンテキストスイッチ (`/proc/stat`)
- `intr` 行の最初の数値を累積割り込み数としてパースします。
- `ctxt` 行の数値を累積コンテキストスイッチ数としてパースします。
- サンプリング間隔の秒数 $t$ に対し、$\frac{\text{現在の累積値} - \text{過去の累積値}}{t}$ を秒間レートとして算出します。

## 4. クロスプラットフォーム仕様 (sysinfoの適用)

Linux以外のプラットフォーム（Windows, macOSなど）では、`sysinfo` クレートを使用してシステムメトリクスを取得します。

- **プロセス状態 (`r`, `b`)**: 取得したプロセスリストから、状態が `ProcessStatus::Run` のものを `r`、`ProcessStatus::UninterruptibleDiskSleep` のものを `b` としてカウントします。
- **メモリ情報 (`swpd`, `free`)**: `total_swap() - free_swap()` および `free_memory()` から算出します。`sysinfo` では詳細な `buff`, `cache` の分割取得が限定されるため、Linux以外の環境ではこれらを `0` (非活性灰色表示) として扱います。
- **システム情報 (`in`, `cs`)**: `sysinfo` では取得できないため、`0` として扱います。
- **CPU 使用率**: 全体CPU使用率 `cpu_usage()` を `us` にマッピングし、アイドル率を `100 - cpu_usage` として `id` に表示します。`sy` および `wa` は `0` として出力します。

## 5. カラー表示の判定基準と ANSI コード

端末上での情報視認性を最大化するため、以下の条件に該当する値を出力する際、標準の 8-bit ANSI エスケープコードを付与します。

| 対象カラム | 判定条件 | 適用スタイル | ANSI シーケンス |
| :--- | :--- | :--- | :--- |
| **すべて** | `値 == 0` | 灰色 (非活性表示) | `\x1b[90m` |
| **swpd** | `swpd > 128MB` | 赤色太字 (スワップ過多) | `\x1b[1;31m` |
| **free** | `free < 512MB` | 赤色太字 (メモリ枯渇) | `\x1b[1;31m` |
| | `free < 1536MB` | 黄色太字 (メモリ警告) | `\x1b[1;33m` |
| **cs** | `cs > 2000` | 黄色太字 (コンテキストスイッチ過多) | `\x1b[1;33m` |
| | `上記以外かつ値 > 0` | 緑色 (アクティビティ) | `\x1b[32m` |
| **in** | `値 > 0` | 緑色 (アクティビティ) | `\x1b[32m` |
| **us** | `us > 80%` | 赤色太字 (CPU高負荷) | `\x1b[1;31m` |
| | `us > 40%` | 黄色太字 (CPU警告) | `\x1b[1;33m` |
| **sy** | `sy > 40%` | 赤色太字 (カーネル負荷高) | `\x1b[1;31m` |
| | `sy > 20%` | 黄色太字 (カーネル警告) | `\x1b[1;33m` |
| **wa** | `wa > 15%` | 赤色太字 (I/Oボトルネック) | `\x1b[1;31m` |
| **timestamp**| 常に適用 | 青色 (タイムトラック) | `\x1b[34m` |

## 6. 二重起動防止仕様 (Windows)

Windows環境における多重起動を防止するため、プロセス開始時に Windows Named Mutex を用いたインスタンスロックを実行します。

- **ミューテックス名**: `Local\MyVMSTAT-single-instance-mutex`
- **動作フロー**:
  1. `CreateMutexW` を呼び出して名前付きミューテックスを生成。
  2. 生成に失敗するか、`GetLastError()` が `ERROR_ALREADY_EXISTS` を返した場合、すでに別インスタンスが稼働中と判断。
  3. 標準エラー出力にエラーメッセージを出力し、プロセスを `exit(1)` で異常終了させる。
  4. 正常に生成できた場合は、ミューテックスのハンドルをプロセス存続中リークさせて保持（プロセス終了時にOSが自動で回収）。

## 7. コマンドライン引数とオプション仕様

アプリケーションは起動時の引数を解析し、以下の動作を行います。

### 7.1 引数の形式
```bash
MyVMSTAT [delay [count]]
MyVMSTAT -h | --help
MyVMSTAT -v | --version
```

### 7.2 オプション説明
- **`-h, --help`**:
  アプリケーションの概要、使用方法、コマンドライン引数、各カラムの着色閾値（カラー凡例）のヘルプメッセージを標準出力に出力し、プロセスを正常終了します。
- **`-v, --version`**:
  `MyVMSTAT version [バージョン番号]` を標準出力に出力し、プロセスを正常終了します。
  - **バージョン番号の取得**: Rust の `env!("CARGO_PKG_VERSION")` を使用し、ビルド時に `Cargo.toml` の `version` フィールドから動的に取得します。ソースコードへのハードコーディングは行いません。

### 7.3 引数バリデーション
- `delay` に正の数値以外（文字列や負の数）が指定された場合、エラーを表示して使用方法を提示し、終了コード `1` で異常終了します。
- `count` に正の整数以外（文字列や負の数、小数）が指定された場合、エラーを表示して使用方法を提示し、終了コード `1` で異常終了します。
- 引数の総数が 3 を超える場合、「Too many arguments.」エラーを表示して終了コード `1` で異常終了します。
- 不正なハイフンオプション（例：`--invalid`）が指定された場合、「Invalid option」エラーを表示して終了コード `1` で異常終了します。
